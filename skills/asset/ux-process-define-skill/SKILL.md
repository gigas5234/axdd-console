# UX Process Define

## 🎯 Purpose
UX 어셋 (체크리스트·예시·룰) 풀세트를 기반으로 **도메인 특화 UX 프로세스 플랜**을 정의한다. UX 트랙의 시작점.

## 📥 Input
- `ui_ux_requirement_summary.md` (Step 1 출력 — UX 관련 요구사항)
- `assets/ux-process-asset-pack.md` (UX 어셋 풀세트)
- `references/double-diamond-method.md`

## 📤 Output
- `ux_process_plan.md` — 도메인 특화 Double Diamond 5주 플랜 + 페르소나 3종 + 단계별 액션 표

## 🔧 동작
1. 도메인 특화 페르소나 3종 작성 (예: 헬스케어 → 환자/보호자/의료진)
2. Double Diamond 4단계별 액션 표 (Discover / Define / Design / Validate)
3. 단계별 산출물 정의 (interview / persona / flow / IA / test report 등)

## ✅ Validation
- 페르소나 ≥ 3종 (Persona × Goal × Pain × Insight)
- Double Diamond 4단계 모두 정의됨
- 각 단계별 액션 표에 Method · Output 명시
- 도메인 키워드 ≥ 10회 등장

## 📚 Assets
- `assets/ux-process-asset-pack.md` — UX 어셋 풀세트 (체크리스트·예시)
- `references/double-diamond-method.md` — 방법론 정의
